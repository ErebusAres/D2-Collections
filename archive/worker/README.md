# D2 Collections Cloud Sync

This Worker adds an optional Cloudflare D1 backend for the static GitHub Pages site.

## What It Does

- `GET /api/snapshots` returns saved Ares/Icee/Fears collection snapshots.
- `POST /api/snapshots` verifies the current Bungie access token, maps the login to Ares, Icee, or Fears, then saves ownership, catalyst, completion, and Rahool resource data.
- `POST /api/auth/exchange` exchanges a Bungie authorization code and stores the refresh token encrypted in D1.
- `POST /api/auth/refresh` rotates Bungie tokens behind an opaque, hashed browser session.
- `DELETE /api/auth/session` revokes the local persistent session.
- The browser never receives D1 credentials, the Bungie client secret, the encryption key, or a stored refresh token.

## Deploy Steps

1. Sign in to Cloudflare and enable Workers.
2. Run `npm install`.
3. Run `npm run cf:d1:create`.
4. Copy `worker/wrangler.toml.example` to `worker/wrangler.toml`.
5. Put the D1 `database_id` from step 3 into `worker/wrangler.toml`.
6. In the Bungie application settings for client `53180`, set **OAuth Client Type** to **Confidential**. Keep the redirect URL as `https://erebusares.github.io/D2-Collections/index.html`.
7. Run `npx wrangler secret put BUNGIE_API_KEY --config worker/wrangler.toml` and paste the D2 Collections Bungie API key.
8. Run `npx wrangler secret put BUNGIE_CLIENT_SECRET --config worker/wrangler.toml` and paste the client secret generated by Bungie.
9. Generate a long random value, then run `npx wrangler secret put OAUTH_ENCRYPTION_KEY --config worker/wrangler.toml` and paste it. Back this value up securely; changing it invalidates stored sessions.
10. Run `npm run cf:d1:migrate`.
11. Run `npm run cf:deploy`.
12. Copy the deployed Worker URL into `data/bungie-config.js` as `cloudSyncApi`.

After deployment, each player signs in once more. The browser stores only an opaque session token. Bungie access tokens are renewed through the Worker, while refresh tokens remain AES-GCM encrypted in D1. Bungie can still revoke authorization or expire a refresh token; those cases correctly require a new login.

Current deployment:

- D1 database: `d2-collections`
- Worker URL: `https://d2-collections-sync.erebusares.workers.dev`

Cloudflare D1 is currently included on Workers Free, with daily read/write limits and a 5 GB total D1 storage allowance. If the free daily limits are exceeded, D1 returns errors until the limit resets.
